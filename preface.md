# 前言

## 编写意图

随着大型语言模型（Large Language Model，LLM）在工业界的广泛应用，推理（Inference）已经从“能跑”变成了“要用得起”。训练一个模型可能只需要几个月，但推理服务要持续运行几年——每一个token的计算、每一次请求的延迟、每一字节的带宽，都在持续消耗资源。

我们编写这本书的初衷，是希望帮助工程师们建立起**系统级的推理优化能力**——不是学会某个框架的使用技巧，而是理解**为什么优化有效、在什么条件下有效、以及如何衡量优化效果**。

## 本书特点

1. **第一性原理出发**：不只讲“怎么做”，更讲“为什么这样做”。理解原理是灵活应用的前提。

2. **工程导向**：每一个技术点都配有生产环境可用的配置示例、监控指标和故障排查指南。

3. **量化思维**：强调用数据说话——优化前后的延迟、吞吐、成本变化，都需要可度量的对比。

4. **场景适配**：不同业务场景的优化重点不同，本书帮助读者建立“判断框架”而非机械套用。

## 适用范围

- **平台工程师 / Infra 工程师**：需要搭建和维护推理平台，关注成本、稳定性与可扩展性
- **后端工程师 / 全栈工程师**：将 LLM 接入业务，关注 API 延迟、失败率与用户体验
- **ML 工程师 / 算法工程师**：关注模型质量与系统约束的平衡
- **技术决策者（CTO、架构师）**：需要评估技术路线与投资回报

## 编写方法

本书采用“问题驱动”的组织方式：每章从实际问题出发，先分析问题根源，再介绍解决方案，最后给出工程实践指南。

各章内容相对独立，读者可以根据自己的需求选择性阅读，也可以按顺序通读形成系统认知。

## 编写分工

本书由多人协作完成，各章节的编写分工如下（按章节顺序）：

- 第1-2章：主编负责整体框架与技术方向把控
- 第3-5章：GPU基础与环境搭建
- 第6-7章：KV Cache与调度策略
- 第8-9章：量化与投机采样
- 第10章：生产部署与运维
- 第11章：高级话题与前沿方向

附录部分由团队共同整理。
