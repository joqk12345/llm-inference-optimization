# 内容摘要

---

## 📖 内容概述

本书系统性地介绍大语言模型（LLM）推理优化的完整技术栈，从优化动机、硬件基础到生产部署与高级专题，为工程师提供一条从“理解瓶颈”到“落地优化”的主线。全书当前分为四大部分、共11章：**动机与路径篇**（第1-2章）回答“为什么推理优化值得做、行业正在往哪里走”；**基础篇**（第3-5章）建立 GPU、运行环境、prefill/decode、Attention 和关键指标等共同前置知识，把后续优化要解决的问题空间搭出来；**核心技术篇**（第6-9章）分别展开 KV 管理与复用、请求调度与预算分配、模型表示如何量化压缩、生成流程如何用投机采样改写；**生产与进阶篇**（第10-11章）先处理生产部署与运行治理，再延伸到 Agent 基础设施、异构硬件、MoE 等更前沿的专题。整体目标不是堆叠术语，而是帮助读者建立一套可迁移的推理优化判断框架。

## ✨ 内容特色

本书有五个核心特点。**第一，按工程决策组织内容**：不是单独讲“一个技巧”，而是把瓶颈、指标、代价、适用边界放在一起讲，方便读者做取舍。**第二，学习路径连续**：从第1章的商业动机与技术全景，一路过渡到第6章的 KV 管理、第7章的请求调度、第10章的生产部署与运行治理，再到第11章的高级专题，减少“知道名词但不会落地”的断层。**第三，案例和代码以教学有效性为先**：当前仓库已提供基础章节相关的可运行环境检查与示例代码，后续章节的配套代码会逐步补齐；正文中的代码块也明确区分“示意代码”和“可直接运行的示例”。**第四，强调成本与可运营性**：推理优化不只看速度，还看 TTFT、吞吐、显存占用、故障恢复、ROI 与长期运维成本。**第五，在线迭代更新**：书稿以在线形式持续修订，允许及时修正结构、案例和术语，使内容逐步收敛为更稳定的工程参考资料。

## 👥 目标读者

本书面向四类核心读者。**AI 工程师和研究员**：如果你理解模型原理，但希望把系统真正跑快、跑稳、跑便宜，第5-11章会给你一条从推理机制、KV 管理、请求调度到生产部署与运行治理及前沿专题的主线。**基础设施工程师**：如果你熟悉 Kubernetes、服务治理和观测体系，但刚进入 AI 推理领域，第3-7章会帮助你理解 GPU、KV 管理、请求调度与尾延迟之间的关系，第10章则从生产部署与运行治理视角给出落地方法。**技术管理者和架构师**：如果你关心选型、预算、容量规划和 ROI，第1-2章提供背景与趋势，第6-10章提供更贴近资源治理、系统取舍与运营闭环的问题框架。**希望系统入门推理优化的开发者**：如果你会 Python，想沿着主线逐章学习，而不是零散看博客，第3-5章是推荐起点。本书不适合完全零基础的新生，也不适合只想快速背 API 或只关心纯理论推导的读者。
